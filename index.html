Why TypeScript with React?

TypeScript has become the standard for React codebases ‚Äî from small startups to large enterprise. It gives you compile-time type checking and excellent editor autocompletion, without slowing down the developer experience. And the learning curve? Much gentler than you think.

1. Props ‚Äî the first win

The moment you type a component's props, you get instant feedback. Forgetting to pass a required prop? TypeScript yells (in a friendly way).

// React + TypeScript component type ButtonProps = { label: string; onClick: () => void; variant?: 'primary' | 'secondary'; // optional with union }; const Button = ({ label, onClick, variant = 'primary' }: ButtonProps) => { return <button className={`btn-${variant}`} onClick={onClick}>{label}</button>; };

variant? means optional. If you mistype "primery", TS will gently correct you. No more runtime guessing.

2. useState and inference

TypeScript often infers types automatically. But with complex state, you can be explicit.

const [user, setUser] = useState<{ name: string; age: number } | null>(null); // user is either null or the object shape
üîπ Pro tip: Use interface for props that are exported or extended; type is great for unions and tuples. Both work.

Event handlers & typings

React provides built-in event types. No need to memorize ‚Äî just hover over an inline handler and copy the type.

const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => { console.log(e.target.value); };

Common patterns in React + TS

children prop

type CardProps = { children: React.ReactNode; title: string; }; const Card = ({ children, title }: CardProps) => ( <div class="card"><h3>{title}</h3>{children}</div> );

React.ReactNode is the most flexible ‚Äî it includes JSX, strings, numbers, fragments, etc.

useReducer with discriminated unions

This is where TypeScript shines. Define actions as a union.

type State = { count: number }; type Action = | { type: 'increment' } | { type: 'decrement' } | { type: 'set'; payload: number }; function reducer(state: State, action: Action): State { switch (action.type) { case 'increment': return { count: state.count + 1 }; case 'decrement': return { count: state.count - 1 }; case 'set': return { count: action.payload }; } }

Migrating an existing React project

You don't have to rewrite everything. Start with .tsx files and add gradual typing. The TypeScript compiler will guide you. Use // @ts-expect-error as a temporary escape hatch, but aim to remove them.

TypeScript doesn't bite ‚Äî it's just JavaScript with superpowers. Your future self (and your team) will thank you.
